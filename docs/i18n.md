# i18n Convention

## Locale files
- `apps/web/public/locales/en/common.json`
- `apps/web/public/locales/de-CH/common.json`

## Key naming
- Use namespaces by domain:
  - `recording.*`
  - `topics.*`
  - `psychologists.*`
  - `common.*`
  - `errors.*`
  - `language.*`

## Rules
- Do not hardcode user-facing copy in feature components.
- Keep key names stable; only update values when wording changes.
- Add keys in both `en` and `de-CH` together in one change.
